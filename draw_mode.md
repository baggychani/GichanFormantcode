# 그리기 모드 기획안 (1차 초안)

> 단일 플롯(일반 모드)에서만 구현. 다중 플롯은 단일에서 완벽히 동작한 뒤 별도로 검토.

---

## 1. 개요

- **목적**: 연구자들이 Formant Plot 위에 선·영역·참조선을 그려 모음 공간·diphthong 등을 표시.
- **범위**: 선(Polyline), 영역(Polygon), 참조선(Reference line) 세 가지 그리기 유형.
- **진입**: 분석 도구의 **그리기** 버튼 클릭 → **선 / 영역 / 참조선** 중 모드 선택 후 캔버스에서 그리기.
└ comment. 모드를 선택한 후에 캔버스에서 그리는 것이 아니라, 모드를 고를 수 있는 indicator를 창 내에 띄우고 무엇을 누르느냐에 따라 그때그때 전환될 수 있게 하는 것이 상급 프로그램의 기본 자세.
---

## 2. 그리기 유형

### 2.1 선 (Line / Polyline)

- **동작**
  - 캔버스 클릭으로 점 추가. 점과 점 사이가 직선으로 연결. (선은 점보다 아래 위치에 있음.)
  - 데이터 좌표(F1-F2, nF1-nF2, F1-F3 등 X축, Y축 데이터를 받아 그에 기반하여)로 저장 → 축 범위/스케일 변경 시에도 비율 유지.
- **스냅**
  - 눈금자 툴과 동일: centroid·raw 점 호버 시 스냅, 툴팁에 좌표 표시. (공용 로직 사용)
  - 스냅 끄기 옵션: 자유 클릭만. → comment : 스냅 끄기는 불가능. 단, tool_ruler에서는 호버될 때 생기는 동그라미 또는 사각형이 결과물로도 표시가 되었는데, 그리기 모드의 선이나 영역에는 원이나 사각형이 결과물로 남지 않음.
- **완료**: **Enter** 키로 현재 선 완료. 완료 시 레이어에 객체로 등록.
- **취소**
  - **Esc**: 지금 그리던 선 전체 취소(점 0개로 초기화).
  - **Ctrl+Z**: 점 하나만 롤백(선 단위 아님).
- **화살표**
  - 선과 동일하게 먼저 선을 그림 → 완료 후 **디자인**에서 "화살표" 스타일 적용.
  - **끝에만 화살표** / **선분마다 화살표** 는 사용자가 디자인에서 선택.
- **레이어 이름**
  - 자동 제안: 꼭지점이 centroid에 스냅됐으면 `"선 : a-b-c"`, 아니면 `"선 1"` 등.
  - 레이어 이름 수정 기능은 현재 라벨 레이어에도 없음 → **공통으로 레이어 이름 편집 기능 추가** 필요.

### 2.2 영역 (Area / Polygon)

- **동작**
  - 선과 동일하게 클릭으로 점 추가.
  - **시작점 근처 다시 클릭** 또는 **"영역 닫기" 버튼/동작**으로 마지막 점을 시작점과 연결해 폐곡선 완성. → 곡선이 아니라 직선이어야 함...
- **채우기**
  - 닫힌 순간 옅은 알파로 채우기(라벨 영역 디자인과 유사). 테두리/채우기 색·스타일은 그리기 디자인에서.
- **레이어 이름**
  - `"영역 : a-b-c-d"` 또는 `"영역 1"` + 사용자 편집.
- **영역 넓이**
  - **로직 필수**: 그린 폴리곤의 넓이를 (데이터 좌표 기준) 계산하는 로직을 반드시 구현.
  - **표시 방식**: UI에 어떻게 노출할지(툴팁, 레이어 행, 별도 패널 등)는 추후 결정. 일단 계산 함수/모듈은 준비.

### 2.3 참조선 (Reference line)

- **동작**
  - 수평선 또는 수직선(또는 2점으로 정의된 직선)을 하나 그리는 모드.
  - 예: "F1 = 500 Hz" 수평선, "F2 = 1500 Hz" 수직선.
- **입력 방식 (후보)**
  - **A**: 축과 평행만 — "수평" 선택 후 한 점 클릭(또는 값 입력)으로 높이 고정.
  - **B**: 2점 클릭으로 직선 하나 정의. 수평/수직은 2점이 겹치는 축으로 자동 인식 가능.
  - 기획 확정 시 하나로 정리.
- **레이어 이름**
  - `"참조선 : F1=500"`, `"참조선 : 수직 F2=1500"` 등 + 사용자 편집.

---

## 3. 레이어 연동

- **그리기 탭** (이미 레이어 도크에 "그리기" 탭 존재)
  - **상단**: 그리기 공통·개별 디자인(선 색/두께/스타일, 영역 채우기, 화살표 유무·위치 등).
  - **목록**: 그린 객체 순서대로 — "선 : a-b-c", "영역 : i-a-u", "참조선 : F1=500" …
  - 각 행: 눈(가시성), 이름, 순서 변경, 삭제, (선택) 잠금.
  - **레이어 이름 수정**: 라벨·그리기 공통으로 "이름 편집" 기능 추가 필요.
- **표시 순서**
  - 그리기 객체 간 순서 변경 가능(드래그 등). 플롯 위에 그려지는 z-order와 연동.

## 4. 키·동작 정리

| 동작           | 선           | 영역              | 참조선   |
|----------------|--------------|-------------------|----------|
| 완료           | **Enter**    | 닫기 후 완료      | 2점 후 완료 등 |
| 취소(전체)     | **Esc**      | Esc               | Esc      |
| 점 롤백        | **Ctrl+Z**   | Ctrl+Z            | -        |

---

## 5. 데이터·좌표

- **저장 단위**
  - 객체 하나 = 선 하나 / 영역 하나 / 참조선 하나.
- **공통 필드(안)**
  - `type`: `"line"` | `"polygon"` | `"reference"`
  - `name`: 표시 이름.
  - `visible`: bool.
  - `order`: 정수(z-order 또는 목록 순서).
  - (선/영역) `points`: 데이터 좌표 배열 `[[x,y], ...]`.
  - (참조선) `mode`: `"horizontal"` | `"vertical"` 등, 및 값 또는 2점.
  - **스타일**: 색, 두께, 선 스타일, 화살표(끝만/선마다), 채우기(영역) 등 — 그리기 디자인 연동.
- **좌표계**
  - 모든 점은 **데이터 좌표**(F1, F2 또는 nF1, nF2)로 저장. 축 스케일 변경 시에도 일관되게 표시.

## 6. 코드 구조 제안

- **폴더**
  - 프로젝트 내 `draw/` (또는 `GichanFormant/draw/`) 생성.
- **모듈 분리**
  - `draw_line.py` — 선(폴리라인) 그리기·저장·표시 로직.
  - `draw_polygon.py` — 영역(폴리곤) 그리기·저장·표시·**넓이 계산** 로직.
  - `draw_reference.py` — 참조선 그리기·저장·표시 로직.
  - 필요 시 `draw_common.py` 또는 `draw_utils.py` — 좌표 변환, 스냅, 데이터 모델 공통 부분.
- 그리기 관련 기능은 위 폴더·별도 py로 유지해 레이어·캔버스와의 연동만 UI에서 처리.

## 7. 영역 넓이

- **요구**
  - 영역(폴리곤)의 넓이를 **데이터 좌표 기준**으로 계산하는 로직 반드시 구현.
- **표시**
  - 툴팁, 레이어 행 부가 정보, 별도 패널 등은 미정. 1차에서는 **계산 로직 및 단위 정의**만 확정.
